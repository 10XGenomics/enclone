enclone variable inventory

This page provides an alphabetical catalog of enclone variables, along with some general information.

General information

All variables are in one of two classes:

  • lvar: a lead variable; these are not associated to a particular chain, and in clonotype tables appear at the beginning (leading), before the variables that are associated to a chain
  • cvar: a chain variable; these are associated to a particular chain. Except when used in CVARS or CVARSP, these need to be suffixed by the chain number.

All variables are specified at one of the following levels:

  • cell: defined for each cell
  • cell-exact: defined for each cell, and also has a value for an exact subclonotype
  • exact: defined for each exact subclonotype
  • clono: defined for each clonotype
  • group: defined for each group of clonotypes.

This table shows how the level is reflected in the output:

level visual output parseable output notes
exact cell exact cell
cell no yes no yes
cell-exact yes yes yes yes parseable-cell output is only for _cell version
exact yes no yes yes
clono yes no yes yes
group no no yes yes

In the PER_CELL mode, where one line is shown per cell, any variable defined at the cell level should show up for each cell, and if not, this is a flaw, and noted below. Variables defined at exact or higher level are not shown for each cell, as this would be redundant.

For variables defined at the cell level, there is a version of the variable name that is suffixed with _cell. This version cannot be used in clonotype tables. It can be used for parseable output, but only if one specifies PCELL.

The following example illustrates some of these points: 

enclone BCR=123085 CDR3=CALMGTYCSGDNCYSWFDPW PER_CELL POUT=stdouth PCELL PCOLS=barcode,u1,u_cell1
