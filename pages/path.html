Setting your path for enclone

This page explains how to edit your shell initialization file to add ~/bin to the beginning of your PATH. This will work in both Linux and Mac environments. If you are using a Windows computer, it will work if you use Windows Subsystem for Linux (WSL) or are using Linux through dual boot.

1. Why is this necessary?

If you do this, then every time you want to run enclone, you'll type enclone. Otherwise you'll have to type ~/bin/enclone every time, and that's inconvenient.

2. Are these instructions universal?

Not exactly. They only work if you're running the bash shell. To determine if this is the case, type echo $SHELL. If the answer is /bin/bash or some other string that ends with /bash, then you are. Otherwise, the instructions on this page will not work. If you're stuck, let us know at enclone@10xgenomics.com, and we'll add further instructions.

3. Now type exactly this. You only need to do it once, as it will persist "forever". 
test -r ~/.bash_profile && echo 'PATH=~/bin:$PATH' >> ~/.bash_profile || echo 'PATH=~/bin:$PATH' >> ~/.profile

This will put ~/bin at the beginning of your path. It is complicated because some versions of Linux use the file .bash_profile, and some use .profile.

To get this to take effect, close and reopen all your windows in which you'll be rerunning enclone.

4. Let us know if this doesn't work.

It is conceivable that the instructions are not configured correctly for all computers.

5. Otherwise, carry on with enclone: you're all set!
